Functions

Functions

Functions are declared using the fn or function keywords. Functions must have a body surrounded by curly braces.

fn foo() { }
function foo() { }

Functions have a return type. If no return type is explicitly specified the return type is inferred

fn foo() {
    return 1; /* return type of foo() is inferred to be int32_t */
}
fn foo() : uint32_t {
    return 1;
}

Function returns must all be of a common type.

fn foo() {
    if(true) {
        return 1;    
    }
    else {
        return 2;
    }
}

/* compiler error: Could not infer common return type from operation's return statements */
fn bar() {
    if(true) {
        return 1;    
    }
    else {
        return false;
    }
}

Functions have parameters which must be explicitly typed

fn foo(x : uint32_t, y : uint32_t) { }

When invoking functions the parameters can be specified by position or name. The style of invocation must not be mixed. There are no optional parameters.

foo(1, 2);
foo(x=1, y=2);
foo(x=1, x=2); /* compile error: parameter x has already been specified */
foo(x=1, 2); /* compile error: mixed positional and named parameters */
foo(1, y=2); /* compile error: mixed positional and named parameters */

Extension methods: the first parameter of a function can be written before the function

let bar = 1;
bar.foo(2);
bar.foo(y=2);
bar.foo(x=1, y=2); /* compile error: parameter x has already been specified */

Functions are polymorphic, meaning that there can be multiple functions with the same name who differ through their signature. The return value type and parameter names are not part of the signature.

fn myfunc() { }
fn myfunc(x : uint32_t) { }
fn myfunc(x : uint32_t, y : uint32_t) { }
fn myfunc(a : uint32_t, y : uint32_t) { } /* compile error: duplicate function 'myfunc'  */
fn myfunc(x : uint32_t, y : uint32_t) : bool { } /* compile error: duplicate function 'myfunc' */

Functions can be generic

fn <T> myfunc(x : T) { }
fn <T> myfunc(x : T) : uint32_t {
    return 10;
}
